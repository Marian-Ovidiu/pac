# pac-modernization-planner

## Quando usarla

Usa questa skill quando devi trasformare la roadmap in backlog operativo, milestone, dipendenze e criteri di done.

## Input minimi

- [`docs/modernization-roadmap.md`](/C:/projects/privati/pac/docs/modernization-roadmap.md)
- vincoli di budget, tempo e accessi

## Workflow

1. Leggi baseline, blocker e dipendenze.
2. Trasforma ogni priorita in epic, task e criteri di done.
3. Evidenzia cosa richiede staging, accesso admin o decisione business.
4. Ordina il lavoro per riduzione del rischio.
5. Produci backlog, milestone e prerequisiti.

## Regole

- non anticipare la UI se runtime e pagamenti non sono stabili
- usa riferimenti file concreti

## Output atteso

- backlog pronto per sprint
- milestone
- dipendenze esterne
